## Statusline 兩行

```
wander洞府 ~$123 · 靈力 76% · 1時20分後回復 · 鼎 Opus 4.8 max [1M] 34%
🌤 【築基期 二層 42%】 · 氣海 3.2M/59.8M · 靈石 2上40中
```

上行為**當下**——所在洞府與其薪火、靈力配額、鼎(當下對話所御 model、其推理力度與上下文用量);下行為**道行**——時辰、境界、氣海、靈石。

| 欄 | 義 |
|----|-----|
| 洞府 ~$X | 今之所在,與其生涯薪火(約數) |
| 靈力 X% | 五時辰配額之餘;將竭則示 ⚠ / ✕ 與回復之時 |
| 鼎 M 力度 [量] % | 當下所御 model、其推理力度(low…max,直書不化)、上下文容量與已用之比;數出 Claude Code,不足則不現 |
| 時辰 emoji | 下行行首,隨十二時辰輪轉(子 🌙 → 亥 🌠) |
| 【境 層 %】 | 境界與層內進度,繫於修為 |
| 氣海 X/Y | 池與涯,**投影值**——含未落盤之新氣 |
| ⚠ 寒月+X | 今若結算即散之數;見則速煉速轉 |
| 靈石 | 行囊所藏,緊湊記法(數+品) |

### 可選段(預設關)

既有段皆可開關;下列四段**預設關**,開了(且有其料)才現:

| 段 | key | 顯示 | 義 |
|----|-----|------|-----|
| repo | `repo` | wander-dao/wander | 洞府所屬 git 倉,列於洞府之左(直書不化) |
| 靈力(週) | `manaWeek` | 週 64% · 3天5時後回復 | 七日配額之餘與回復之期 |
| 疾行 | `haste` | ⚡疾行 | 疾行模式開啟時現 |
| 淬煉 | `temper` | 淬煉 +230/-45 | 本回改動之行數(增 / 刪) |

開關:`wander config set statusline.<段> on|off`(段名大小寫不拘),`wander config list` 觀現況。例:`wander config set statusline.temper on`。

初用示【未入門】,片刻自亮——紀錄自立,無須先行指令。status bar 每五秒自動重繪,閒置亦保持最新。

Statusline **純讀快取,絕不掃描,絕不落盤**;其新鮮,由任一指令養之。投影冪等,常亮無虞。

## 資料三層

| 處 | 性 | 失之 |
|----|-----|------|
| `~/.local/share/wander/` | **永久**:年譜與遊戲之狀 | 修行歸零,不可再生 |
| `~/.cache/wander/stats/` | 衍生快取 | 無害,自重建 |
| `~/.config/wander/stats/config.json` | 己之所設 | 偏好失 |

備份者,`~/.local/share/wander/` 一處而已。Windows 之 `~`,即 `%USERPROFILE%`,佈局同。

## 隱私

承諾唯一:**資料永不離爾之機。** 不通於外,不上傳,不錄行蹤;讀算皆於本機,果只落上表三處。

**或問**:薪火與帳單不符?
**曰**:薪火按官價自 token 推,近似之數;訂閱之下,所示者等值 API 之費,非實扣。欲自訂其價,見[附錄甲](06-指令.md)。

**或問**:多機可合乎?
**曰**:今各自修行,不合。

**或問**:年譜溯及幾時?久存乎?
**曰**:源出 Claude Code 之對話卷宗(`~/.claude/projects/` 之 `.jsonl`)。Claude Code 逾 `cleanupPeriodDays` 之日(預設三十)即刪其舊卷;然 wander 每運,先其刪而錄之於**永久年譜**,故一經收錄,雖原卷後遭刪,亦永存不失——此年譜之用也。惟溯之底,止於爾初運 wander 之日;其前既刪之卷,無從追還。欲留更深者,於 `~/.claude/settings.json` 設 `cleanupPeriodDays`(日數,最少一,越大則 Claude Code 留卷越久),予重建年譜時更深之源;然卷積則耗碟,權衡而設。

**或問**:快取可刪乎?
**曰**:隨刪隨建;不可再生者,唯永久一層。
